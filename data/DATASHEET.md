### Datasheets for Datasets Framework [Gebru et al., 2021]

## Motivation

**For what purpose was the dataset created?**

This dataset was created to support comparative evaluation of two LLM-based simulated-learner
prompting protocols (Teachable Agent and Student Simulation) against two baselines, using
matched public tutoring-dialogue scenarios. The dataset provides: (1) 4,800 AI-generated
tutoring dialogues across 4 LLM families, 4 conditions, 3 seeds, and 100 scenarios;
(2) 1,280 Bridge cross-dataset replication dialogues; (3) 400 protocol-element ablation
dialogues; (4) Human coder ratings for 240 dialogues on 9 framework dimensions;
(5) LLM-as-judge ratings for all 4,800 dialogues; (6) Automatic trace metrics.

## Composition

**What do the instances represent?**

Each instance is a structured JSON file representing one AI-generated tutoring dialogue,
containing: scenario metadata, condition label, model identifier, seed, per-turn content
(tutor turns and LLM-generated learner turns), exclusion flag, and token counts.

**How many instances are there?**

| Split | Description | n |
|-------|-------------|---|
| main | Primary MathDial-based dialogues | 4,800 |
| bridge | Bridge cross-dataset replication dialogues | 1,280 |
| ablations | Protocol-element ablation dialogues | 400 |
| Total | | 6,480 |

**Does the dataset contain all possible instances or is it a sample?**

This is a sample: 100 scenarios drawn from MathDial (2,861 total), 80 from Bridge (~700 expert-tutor).
Sampling was stratified by error_type and topic (see `data/scenarios.csv` for sampling details).

**Is there a label or target associated with each instance?**

Each dialogue carries:
- `condition`: C1 (Teachable Agent), C2 (Student Simulation), C3 (Generic), C4 (No-role)
- `model_id` and `model_tag`: which LLM generated the learner turns
- `exclusion_flag` and `exclusion_code`: whether excluded per pre-registered rules E1–E6

Human-coded dialogues (240) carry ratings on TA1–TA4, SS1–SS5, use_case_decision, and 6 failure flags.

**Is any information missing from individual instances?**

Excluded dialogues (flag=True) are retained but should not be used for analysis.
System prompts are retained for reproducibility but should not be shared with coders.
Original MathDial/Bridge human data is NOT included (governed by original licences).

**Are relationships between instances made explicit?**

Yes. `scenario_id` links instances across conditions and seeds. `seed` allows tracking
generation variability. `packet_id` links human-coded instances to reliability data.

**Are there recommended data splits?**

- Confirmatory analysis: human-coded 240-dialogue subset only
- Generalization analysis: full 4,800-dialogue set with judge ratings
- Cross-dataset analysis: bridge split only

**Are there any errors, sources of noise, or redundancies?**

- API errors: some turns contain empty strings (E1 exclusion); retained in JSON for transparency
- Token truncation: turns occasionally hit max_tokens=600; flagged in per-turn records
- Seed variability: 3 seeds produce 3 replicates per scenario × condition × model; all retained

**Is the dataset self-contained, or does it link to or rely on external resources?**

The dataset contains only AI-generated content. It does NOT contain original MathDial or
Bridge human transcripts. Users must separately download those datasets (see below).

**Does the dataset contain data that might be considered confidential?**

No. All content is AI-generated and contains no personal information.
Human coder annotations are anonymised at coder level (coder_id = "C_A", "C_B", "adjudicator").

**Does the dataset contain data that might be considered offensive, insulting, or threatening?**

All dialogues address 7th-grade math topics (fractions, algebra, word problems).
No offensive content expected; occasional API refusal responses are excluded (E3).

## Collection Process

**How was the data associated with each instance acquired?**

Learner turns were generated by calling LLM APIs (OpenAI, Anthropic, Google, HuggingFace)
with frozen prompt protocols (v1.0, commit c2c41e6). Tutor turns are replayed verbatim from
the source dataset (MathDial / Bridge), as per Decision D2.

**What mechanisms or procedures were used to collect the data?**

`src/generation/runner.py` — multi-model dialogue generation runner. Supports checkpoint-resume,
rate limiting, cost logging, and per-dialogue JSON serialisation.

**Over what timeframe was the data collected?**

Data collection is planned for Weeks 2–5 of the 16-week study timeline (May–June 2026).

**Were any ethical review processes conducted?**

IRB exempt determination filed with [INSTITUTION], Protocol [NUMBER].
No human participants; all source data is pre-existing and publicly consented.

**Did you collect the data from the individuals in question directly, or obtain it via third parties?**

LLM-generated; no direct human data collection. Source scenarios derived from MathDial
(Macina et al., 2023, CC-BY-4.0) and Bridge (Wang & Demszky, 2024, CC-BY-NC-4.0).

## Preprocessing / Cleaning / Labeling

**Was any preprocessing/cleaning/labeling of the data done?**

Yes:
- Exclusion rules E1–E6 applied (pre-registered); flagged dialogues retained
- Condition leakage check via `src/coding/packetize.py` (automatic phrase redaction)
- Hash anonymisation for coder packets

**Was the "raw" data saved in addition to the preprocessed/cleaned/labeled data?**

Yes. All raw JSON files are retained. Preprocessed files (coder_packets/, coder_ratings_raw.csv)
are generated from raw data and can be regenerated from source.

**Is the software that was used to preprocess/clean/label the instances available?**

## Uses

**Has the dataset been used for any tasks already?**

This is a new dataset created specifically for this study.

**What (other) tasks could the dataset be used for?**

- Benchmarking LLM instruction-following fidelity in educational role-playing
- Training LLM-as-judge models for educational dialogue quality assessment
- Studying prompt robustness across LLM families
- Replicating or extending protocol-level comparison studies

**Is there anything about the composition of the dataset or the way it was collected and
preprocessed/cleaned/labeled that might impact future uses?**

- English-language only; primarily US-curriculum 7th-grade math
- RLHF-aligned models only (all 4 families use RLHF)
- Tutor turns are from human-controlled corpora (not AI-generated tutors)
- Exclusion rate ~3.8% overall; higher for C2 × M4 (~10.4%)

**Are there tasks for which the dataset should not be used?**

- Should NOT be used to train models to simulate real student behaviour without careful validation
- Should NOT be interpreted as representing real student performance or learning outcomes
- Should NOT be used to make claims about model superiority without controlling for protocol
